课题组 GitHub Organization 管理办法（2026.04.10）

一、基本结构

1. 课题组统一使用一个 GitHub Organization 管理全部公共仓库和项目仓库。

2. Organization 仅设 2 名 Owner：课题组负责人和 1 名副管理员。其余人员均为 Member。

3. Organization 固定设置 4 个 team：all-members、team-traj、team-vr、team-vis。

4. 每名成员除加入 all-members 外，只能固定属于 team-traj、team-vr、team-vis 三者之一，不再建立项目 team。

5. 每个 team 原则上设置 2 名 maintainer。

二、权限规则

1. Organization 的 base permission 统一设置为 none。

2. 公共仓库统一授予 all-members 的 write 权限，仓库维护人授予 admin 权限。

3. 小组项目仓库统一授予对应 team 的 write 权限。

4. 每个重要项目仓库原则上设置 2 名 admin。

三、新仓库管理

1. 新仓库创建后，应立即明确仓库类别（公共、小组项目）、完成对应 team 授权、指定仓库 admin，并补充仓库用途与维护说明。

四、成员变动

1. 新成员加入时，依次加入 Organization、all-members 和所属 team。

2. 成员离组时，按照“先交接、后撤权”原则处理：先完成组内职责交接和文档交接，再撤销仓库权限、移出 team、移出 all-members，最后移出 Organization。

五、附则

1. 本办法遵循组织级集中管理、成员单组归属、固定组长期稳定、仓库权限显式配置、先交接后撤权的原则。

2. 本办法自发布之日起试行，由课题组负责人负责解释和修订。
